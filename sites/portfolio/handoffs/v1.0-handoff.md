## Design Intent

### Theme — "Terminal Meets Modern"

Dark theme inspired by Ruben's self-hosted Dashboard. Security engineer aesthetic — not flashy, not corporate. Terminal references without being gimmicky.

|-------|-----|-------|

| `--color-surface-2` | `#232736` | Nested surfaces, tech badges |
| `--color-border` | `#2e3347` | Borders, dividers |
| `--color-text` | `#e4e6f0` | Primary text |

| `--color-yellow` | `#ffd93d` | Warnings, featured badges |

- **Headings + Body:** Inter (400-900)
- **Code/Terminal:** JetBrains Mono (400-600)
- Loaded via Google Fonts (switch to self-hosted before production)

- **Grid background:** Subtle 60px grid pattern on hero section
- **Glow orbs:** Animated blur orbs behind hero content
- **Gradient text:** `linear-gradient(135deg, accent, green)` on key headlines
- **Terminal preview:** Fake terminal in hero showing services
- **Framer Motion:** Fade-up animations on scroll, spring-animated nav indicator

### Icons

Lucide React throughout. Key icons: `Shield`, `GitBranch`, `Search`, `Cloud`, `Terminal`, `Lock`, `Server`, `Code`.

## Tool Setup — Required MCP Servers & CLIs

### 1. shadcn/ui MCP Server

The official shadcn MCP server lets Claude Code browse, search, and install components directly from registries using natural language.

**Install in Claude Code config (`.claude/settings.json` or project-level):**
```json
{
  "mcpServers": {
    "shadcn": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/shadcn-mcp@latest"]
    }
  }
}
```

**Or add via CLI:**
```bash
claude mcp add shadcn -- npx -y @anthropic-ai/shadcn-mcp@latest
```

**Usage:** Once installed, Claude Code can run `npx shadcn@latest add button` or install from 21st.dev registries using `@21st/component-name` syntax.

### 2. 21st.dev Registry

21st.dev is a community registry of shadcn/ui-based components. After initializing shadcn, configure the 21st.dev registry:

**In `components.json`:**
```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": true,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "src/app/globals.css",
    "baseColor": "zinc",
    "cssVariables": true
  },
  "registries": {
    "21st": {
      "url": "https://21st.dev/r"
    }
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  }
}
```

**Browse components at:** https://21st.dev/community/components
**Install syntax:** `npx shadcn@latest add "@21st/component-name"`

### 3. Google Stitch + stitch-mcp

Stitch-mcp connects Google Stitch (AI design tool) to Claude Code. Use it to generate polished UI designs, then pull the HTML/CSS into the project.

**Install:**
```bash
npm install -g stitch-mcp
```

**Configure MCP server:**
```json
{
  "mcpServers": {
    "stitch": {
      "command": "stitch-mcp",
      "args": ["--project-id", "<STITCH_PROJECT_ID>"]
    }
  }
}
```

**Or add via CLI:**
```bash
claude mcp add stitch -- stitch-mcp --project-id <STITCH_PROJECT_ID>
```

**Workflow:**
1. Design screens in Google Stitch (https://stitch.google.com)
2. Use `build_site` tool to pull all screen HTML into the project
3. Use `get_screen_code` to grab individual screen HTML
4. Use `get_screen_image` to get screenshot references for design fidelity
5. Claude Code converts the Stitch HTML into Next.js components using the existing design tokens

**Recommended Stitch screens to design:**
- Hero section with terminal mockup
- Project showcase cards
- Contact form layout
- Mobile navigation drawer

## Build Tasks — What Claude Code Should Do

Execute in this order. Each task maps to the pipeline stages.

### Task 1: Setup — Install & Initialize (Setup stage)

```bash
cd ~/projects/portfolio
npm install
npx shadcn@latest init
```

**During `shadcn init`:**
- Style: New York
- Base color: Zinc (we override with custom vars anyway)
- CSS variables: Yes
- React Server Components: Yes
- Tailwind CSS path: `src/app/globals.css`
- Components alias: `@/components`
- Utils alias: `@/lib/utils`

**Then verify:**
```bash
npm run dev
# Open http://localhost:3000 — should render the home page
```

**Gate:** Site builds and renders without errors on `localhost:3000`.

### Task 2: shadcn + 21st.dev Components (Build stage)

Install base shadcn components needed across the site:

```bash
npx shadcn@latest add button
npx shadcn@latest add badge
npx shadcn@latest add card
npx shadcn@latest add separator
npx shadcn@latest add tooltip
```

Then browse https://21st.dev for enhanced components to replace/upgrade:

**Priority 21st.dev components to evaluate:**
- Hero section components (animated backgrounds, gradient text)
- Project/bento card layouts
- Animated navigation bars
- Contact form components
- Terminal/code block components
- Tech stack/skill badge grids

**Install from 21st.dev registry:**
```bash
npx shadcn@latest add "@21st/component-name"
```

**Integration rule:** When replacing a section with a 21st.dev component, keep the existing data layer (`src/data/projects.ts`) and wire the new component to consume the same data. Do NOT rewrite the data model.

### Task 3: Stitch Design Polish (Build stage)

If a Stitch project is set up:

1. Create designs in Google Stitch for each page
2. Use `stitch-mcp` to pull the design HTML
3. Extract design patterns (spacing, border-radius, shadow values) into the existing CSS variables
4. Rebuild components to match the Stitch designs using shadcn/21st.dev primitives
5. Do NOT use raw Stitch HTML in production — always convert to proper React components

**If no Stitch project exists yet:**
- Skip this task
- Proceed with the existing scaffolded components
- Stitch polish can be a v1.1 task

## Coding Conventions

- **React:** Functional components with hooks. TypeScript strict. `"use client"` only on interactive components.
- **Styling:** Tailwind utility classes only. Custom CSS vars in `globals.css` for theme tokens. No inline styles, no CSS modules.
- **Components:** shadcn/ui primitives in `components/ui/`. Page sections in `components/sections/`. Data in `data/`.
- **Animations:** Framer Motion for scroll-triggered and layout animations. Keep animations subtle — fade/slide, no bouncing.
- **Icons:** Lucide React only. Import individually: `import { Shield } from "lucide-react"`.
- **File naming:** kebab-case for files. PascalCase for component exports.
- **Imports:** Use `@/` path alias for all imports.

## What NOT to Do

- NEVER remove or restructure `src/data/projects.ts` — all components depend on it
- NEVER change the color tokens without updating all components that reference them
- NEVER add a CMS or backend — this is a static site, data lives in code
- NEVER use raw Stitch HTML in production — always convert to React components
- NEVER add authentication or user accounts — this is a public portfolio
- NEVER add analytics scripts without Ruben's explicit approval
- NEVER use `WidthType.PERCENTAGE` in any table-related component (breaks rendering)
