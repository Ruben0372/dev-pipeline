# Phase 2 Handoff: Issue Reporting System

**Date:** 2026-03-26
**Issues:** #12, #13
**Status:** Complete

## What Was Done

### 2.1 Report Format & Template
- Created `templates/issue-report.md` with full YAML frontmatter schema: id, severity, domain, status, reporter, source_context, project, filed_at, dispatch_mode, assigned_to, resolved_at, dispatch_id
- Added `issues` category to `config/categories.json` (order: 11, icon: alert-circle)
- Created `config/dispatch-rules.json` mapping severity to dispatch behavior:
  - critical/high: auto dispatch, auto approval, branch+PR fix strategy
  - medium/low: queued dispatch, needs human approval
  - Max 3 retries, 15-min cooldown, FIFO with priority interrupt

### 2.2 Shell Script & Auto-Push
- Created `scripts/report-issue.sh`: accepts project, severity, domain, title, body; generates YAML frontmatter; writes file to `sites/{project}/issues/`; commits
- Script auto-detects dispatch mode from `dispatch-rules.json` using Python
- Updated `.githooks/post-commit` to push current branch (not just main)
- Git hooks path already configured (`.githooks`)
- Tested full flow: script creates proper issue file, commits, and post-commit hook pushes

### 2.3 CLAUDE.md Convention
