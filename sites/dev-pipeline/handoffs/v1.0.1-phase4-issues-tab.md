# Phase 4 Handoff: Tower Issues Tab

**Date:** 2026-03-26
**Issue:** #14
**Status:** Complete

## What Was Done

### 4.1 Backend — Frontmatter Parsing + API Endpoints
- Created `IssueRecord` model (`api/internal/models/issue.go`)
- YAML frontmatter parser: extracts all 12 frontmatter fields from issue markdown files
- Added to `SiteRecordStore`: `ListIssues(projectID)`, `ListAllIssues()`, `IssueStats()`
- Created `api/internal/handlers/issues.go` with 3 endpoints:
  - `GET /api/sites/{slug}/issues` — issues for one project
  - `GET /api/issues` — cross-project aggregation
  - `GET /api/issues/stats` — counts by severity, status, project
- Routes wired into `main.go`

### 4.2 Frontend — Kanban Board
- Created `IssueCard.tsx` — severity badge (red/orange/yellow/gray), domain tag, age, project
- Created `IssueBoard.tsx` — 4-column Kanban: Open | Dispatched | In Progress | Resolved
- Modified `SiteRecordTabs.tsx` — detects `issues` category, renders IssueBoard instead of record list
- WebSocket auto-refresh on `issue_discovered`, `issue_dispatched`, `dispatch_completed`, `site_records_updated`
- Stats bar showing total count + critical/high highlights
- Added `IssueRecord` and `IssueStats` types to `types/index.ts`
