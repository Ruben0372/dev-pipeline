## Overview

v1.1.0 transforms the Dashboard from a functional command center into an interactive workspace. It adds 15 features across 6 domains: animations, rich text editing, drawing, weather, productivity timers, and enhanced state management. The release also introduces dev-pipeline integration (v1.1.1) to connect Claude Code agent output directly into The Tower's site records UI.

## Tech Decisions

| Library | Version | Purpose | Rationale |
|---------|---------|---------|-----------|
| **Motion** | 11.x | Animations and transitions | Industry standard for React animations. Renamed from Framer Motion. Layout animations, AnimatePresence for route transitions, spring physics. |
| **BlockNote** | @blocknote/react + @blocknote/core + @blocknote/mantine | Rich text editor | Block-based editor with slash commands, collaborative-ready architecture. Mantine dependency for UI components. Lazy-loaded -- only fetched on /notes route. |
| **perfect-freehand** | 1.x | Freehand drawing | Lightweight (<5KB), produces smooth pressure-sensitive strokes from pointer events. No canvas dependency -- outputs SVG path data. |
| **Open-Meteo** | REST API | Weather data | Free, no API key required, no rate limit concerns for single-user dashboard. Returns JSON with hourly/daily forecasts. |
| **react-timer-hook** | 3.x | Pomodoro timer | Lightweight hook-based timer with useTimer/useStopwatch. No UI opinions -- pairs with existing design system. |
| **Zustand** | 5.x | State management | Minimal boilerplate, TypeScript-first, persist middleware for localStorage. Replaces prop drilling and scattered local state. |

### Bundle Strategy

BlockNote and Mantine add approximately 2MB to the bundle. To prevent regression:

```
manualChunks: {
  'blocknote': ['@blocknote/core', '@blocknote/mantine', '@blocknote/react'],
  'mantine': ['@mantine/core'],
}
```

BlockNote chunk loads only when visiting /notes. All other routes remain under the v1.0.1 bundle size.

## Key Architectural Decisions

### 1. Zustand Over Context API

Zustand was chosen over React Context for global state because Context triggers re-renders on all consumers when any value changes. Zustand's selector pattern prevents unnecessary re-renders. The persist middleware also provides localStorage sync without custom code.

### 2. BlockNote Over Prosemirror/TipTap Direct

BlockNote provides a higher-level block abstraction over Prosemirror. The slash command menu, drag handles, and block types are built in. The tradeoff is the Mantine UI dependency (~800KB), mitigated by code splitting.

### 3. Client-Side Weather Fetching

Open-Meteo is called directly from the browser rather than proxied through the Go API. Rationale: no API key to protect, no server-side caching benefit for a single-user dashboard, and it avoids adding HTTP client code to the Go backend for a non-critical widget.

### 4. SVG Drawing Storage Over Canvas Bitmap

perfect-freehand outputs stroke geometry as point arrays, rendered as SVG paths. Storing as SVG (not PNG/canvas bitmap) preserves resolution independence, enables future editing, and keeps file sizes small (<50KB per drawing).

### 5. Lazy Loading for BlockNote

BlockNote and its Mantine dependency are isolated in a Vite manual chunk. React.lazy() wraps the notes page so the 2MB chunk only loads when the user navigates to /notes. Every other route loads at v1.0.1 speeds.

## Risks and Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| BlockNote bundle bloat breaks load times | HIGH | manualChunks code splitting. Verify with `vite build --report`. BlockNote loads only on /notes. |
| Mantine CSS conflicts with Tailwind | MEDIUM | Scope Mantine styles to BlockNote wrapper. Use Mantine's unstyled mode where possible. Test visual regression on all pages. |
| Motion animations cause layout shifts | LOW | Use layout animations sparingly. Test with `prefers-reduced-motion` media query -- disable animations when set. |
| Open-Meteo API downtime | LOW | Cache last successful response in localStorage. Show stale data with "last updated" timestamp. Widget is non-critical. |
| Zustand migration breaks existing state | MEDIUM | Migrate one store at a time. Keep existing useState as fallback until each store is verified. Run full test suite after each migration. |
| manualChunks misconfiguration causes blank screen | HIGH | Verify chunk loading with `vite preview` before deploying. Test all routes in Docker build. Known issue from v1.1.0 -- see retro. |
| Drawing SVGs accumulate disk space | LOW | Single-user dashboard. At <50KB per drawing, 1000 drawings = 50MB. Add cleanup UI in v1.2 if needed. |
