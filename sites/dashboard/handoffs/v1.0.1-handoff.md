## Detailed Fix Specifications

### Fix 1: FTS5 Notes UPDATE Crash

**Problem:** `PUT /api/notes/{slug}` crashes because the SQL for updating the FTS5 virtual table (`notes_fts`) is incorrect. The UPDATE path tries to delete + re-insert into the FTS5 table but the SQL references wrong columns or uses incorrect syntax.

**Where:** `api/internal/models/note_store.go` (or wherever the note update SQL lives)

**Fix:**
- Locate the UPDATE function for notes
- The correct FTS5 update pattern is: delete the old FTS5 row by rowid, then insert the new one
- Use `DELETE FROM notes_fts WHERE rowid = ?` then `INSERT INTO notes_fts(rowid, title, content) VALUES (?, ?, ?)`
- Alternatively, use FTS5's built-in `UPDATE` syntax: `INSERT INTO notes_fts(notes_fts, rowid, title, content) VALUES('delete', ?, ?, ?)` followed by a regular INSERT

**Test:** Write a dedicated FTS5 integration test that exercises the full lifecycle:
```
create note Б├▓ search (found) Б├▓ update note Б├▓ search (finds updated content) Б├▓ delete note Б├▓ search (not found)
```

### Fix 2: Replace `init()` Handler Pattern with DI

**Problem:** Handlers are defined as package-level `var` in `handlers/health.go` pointing to `notImplemented`, then reassigned in `init()` functions in each handler file. This is fragile, hard to test (can't inject mocks), and Go-unidiomatic.

**Where:** `api/internal/handlers/` Б─■ all handler files

**Current pattern (DISCARD):**
```go
// handlers/health.go
var ListProjects http.HandlerFunc = notImplemented
var GetProject http.HandlerFunc = notImplemented

// handlers/projects.go
func init() {
    ListProjects = listProjects
    GetProject = getProject
}
```

**New pattern (USE THIS):**
```go
// handlers/handlers.go Б─■ define the struct
type Handlers struct {
    db       *sql.DB
    notion   *services.NotionService
    calendar *services.CalendarService
    tasks    *models.TaskStore
    notes    *models.NoteStore
    dispatch *models.DispatchStore
    activity *models.ActivityStore
    hub      *services.Hub
}

func New(db *sql.DB, notion *services.NotionService, ...) *Handlers {
    return &Handlers{db: db, notion: notion, ...}
}

// handlers/projects.go Б─■ methods on the struct
func (h *Handlers) ListProjects(w http.ResponseWriter, r *http.Request) {
    // use h.notion, h.db, etc.
}
```

**main.go update:**
```go
h := handlers.New(db, notionSvc, calSvc, taskStore, noteStore, dispatchStore, activityStore, hub)
r.Get("/api/projects", h.ListProjects)
r.Get("/api/projects/{id}", h.GetProject)
// ... etc
```

**Impact:** This touches every handler file and main.go route registration. Do this FIRST before other fixes since it restructures the handler layer.

**Agent coordination note:** This is a shared-file-heavy change. Do NOT parallelize this with other fixes Б─■ run it as a single sequential task.

### Fix 3: Remove Simulated Dispatch Processing

**Problem:** `api/internal/services/dispatch.go` (or models) has a fake 1-second `time.Sleep` simulating "processing" when a dispatch job is queued. This makes the UI show a progress state that doesn't reflect real work.

**Where:** `api/internal/services/dispatch.go` or `api/internal/models/dispatch.go`

**Fix:** Two options depending on ambition:
- **Option A (minimal):** Remove the `time.Sleep`. Set dispatch status to `completed` immediately with a message like "Dispatch queued Б─■ real execution not yet wired." Update the frontend dispatch status card to show this honestly.
- **Option B (real):** Wire dispatch to actually execute something (e.g., write to a file in `/data/dispatch/`, call a webhook, or create a scheduled task). This is v2 scope Б─■ go with Option A for v1.0.1.

### Fix 4: Bundle IBM Plex Mono Locally

**Problem:** `web/src/index.html` or `web/src/index.css` loads IBM Plex Mono from `https://fonts.googleapis.com`. Self-hosted dashboard behind Tailscale shouldn't have external CDN dependencies.

**Fix:**
1. Download IBM Plex Mono woff2 files (Regular, Medium, SemiBold, Bold Б─■ 400/500/600/700)
2. Place in `web/public/fonts/` (or `web/src/assets/fonts/`)
3. Create `@font-face` declarations in a CSS file or in `index.css`
4. Remove the Google Fonts `<link>` tag
5. Verify the font loads correctly in Docker (nginx serves `/fonts/` path)

### Fix 5: `.env` Validation at Startup

**Problem:** If `.env` has wrong Notion DB IDs or expired Google credentials, the server starts successfully but all API calls fail silently.

**Where:** `api/cmd/server/main.go` or `api/internal/config/config.go`

**Fix:**
- After loading config, before starting the HTTP server:
  1. Make a test query to Notion API: `POST /v1/databases/{NOTION_PROJECTS_DB}/query` with `page_size: 1`
  2. Make a test query to Google Calendar API: `GET /calendars/{GOOGLE_CALENDAR_ID}`
  3. If either fails, log the error clearly and exit with a non-zero code
- Add a `--skip-validation` flag for local dev when APIs aren't available
